## 二、全局规则

### 2.1 数据字典（核心实体）

**Account（账户）**
| 字段             | 类型      | 必填 | 规则                                           |
| ---------------- | --------- | ---- | ---------------------------------------------- |
| account_id       | String    | 是   | 系统生成UUID                                   |
| category         | Enum      | 是   | bank/stock/fund/property/insurance/debt/custom |
| institution_name | String    | 是   | 1-30字符，用户自由输入                         |
| alias            | String    | 否   | 0-20字符                                       |
| icon             | String    | 否   | Emoji，默认按类型取                            |
| sub_type         | Enum      | 否   | 如银行：savings/credit                         |
| is_liability     | Boolean   | 是   | 是否计入负债侧                                 |
| current_amount   | Decimal   | 是   | 见金额规则                                     |
| total_invested   | Decimal   | 否   | 累计投入（投资类）                             |
| currency         | String    | 是   | 默认CNY                                        |
| status           | Enum      | 是   | active/archived                                |
| created_at       | Timestamp | 是   | 系统生成                                       |

**Snapshot（资产快照）**
| 字段         | 类型    | 必填 | 规则             |
| ------------ | ------- | ---- | ---------------- |
| snapshot_id  | String  | 是   | 系统生成         |
| account_id   | String  | 是   | 关联账户         |
| amount       | Decimal | 是   | 该时点金额       |
| record_date  | Date    | 是   | 默认当天，可回溯 |
| note         | String  | 否   | 0-50字符         |
| milestone_id | String  | 否   | 关联里程碑       |

**Milestone（里程碑）**
| 字段          | 类型    | 必填         |
| ------------- | ------- | ------------ |
| milestone_id  | String  | 是           |
| title         | String  | 是（1-20字） |
| date          | Date    | 是           |
| amount_impact | Decimal | 否           |
| icon          | String  | 否           |

### 2.2 金额输入通用规则
| 规则     | 说明                                           |
| -------- | ---------------------------------------------- |
| 数值范围 | 0 ≤ x ≤ 999,999,999.99                         |
| 小数位   | 最多2位，超出自动截断                          |
| 格式化   | 千分位展示，输入时实时格式化                   |
| 负值处理 | 不允许直接输入负号；负债通过`is_liability`标识 |
| 空值校验 | 必填项为空时，"确认"按钮置灰                   |
| 异常输入 | 非数字字符过滤；前导0去除                      |

### 2.3 净资产计算逻辑
```
总资产 = Σ(is_liability=false 账户的 current_amount，按本位币折算)
总负债 = Σ(is_liability=true  账户的 current_amount)
净资产 = 总资产 − 总负债
```

## 三、功能模块 PRD

### 模块1：添加账户【P0】

#### 1.1 流程图
```
[选择资产类型] → [填写账户信息] → [确认添加] → [返回总览/继续添加]
                       ↑__________修改__________|
```

#### 1.2 详细规则

**Step1 选择类型**
- 9宫格展示8个预设类型 + 自定义
- 点击即进入Step2，无需"下一步"按钮（减少操作）

**Step2 填写信息（差异化字段）**

| 类型     | 必填校验           | 特殊逻辑                                                     |
| -------- | ------------------ | ------------------------------------------------------------ |
| 银行     | 机构名+余额        | 选"信用卡"→金额字段标题变"当前欠款"，`is_liability=true`     |
| 股票     | 券商名+总资产      | 可拆分"现金+持仓"，若两者皆填，校验之和=总资产（误差提示，不强制） |
| 基金     | 平台名+总市值      | 类型分布之和不强制等于总市值                                 |
| 固定资产 | 名称+估值          | 可关联负债账户                                               |
| 自定义   | 名称+金额+正负属性 | 用户手动选择计入资产/负债                                    |

**Step3 确认**
- 预览卡片：图标+机构名·别名+金额
- 信用卡/负债类金额显示红色，带"负债"标签
- 「确认添加」后：
  - 创建Account记录
  - 同步创建首条Snapshot（record_date=今天）
  - Toast提示"已添加"，可选"继续添加"或"返回"

#### 1.3 异常处理
| 场景       | 处理                                                       |
| ---------- | ---------------------------------------------------------- |
| 机构名重复 | 允许（同银行多卡场景），但提示"已存在同名账户，是否继续？" |
| 金额=0     | 允许（如新开账户），正常保存                               |
| 中途退出   | 弹窗"放弃添加？"，确认后丢弃                               |
| 网络无关   | 全程本地操作，无网络依赖                                   |

### 模块2：更新资产快照【P0 核心】

#### 2.1 单账户更新（2步）
**入口**：账户列表项右滑/点击"更新"按钮

| 字段     | 规则                                           |
| -------- | ---------------------------------------------- |
| 新金额   | 默认显示上次金额，全选可改                     |
| 记录日期 | 默认今天，可选历史日期（不可晚于今天）         |
| 变化原因 | 选填，提供快捷标签（工资/收益/转账/消费/其他） |

**保存逻辑**：
- 创建新Snapshot
- 更新Account.current_amount = 新金额
- 若同一天已有快照 → **覆盖**当天记录（避免一天多点）

#### 2.2 异常处理
| 场景             | 处理                           |
| ---------------- | ------------------------------ |
| 选择未来日期     | 禁止，日期选择器限制max=今天   |
| 金额与上次相同   | 允许保存（用户主动确认无变化） |
| 历史日期已有快照 | 提示"该日期已有记录，将覆盖"   |
| 批量更新中途退出 | 已确认的不保存，弹窗确认放弃   |

### 模块3：首页总览【P0】

#### 3.1 模块构成（从上到下）
1. **净资产大卡片**
   - 主数字：净资产（大字号）
   - 副信息：较上次 +¥X,XXX (+X.X%)，红涨绿跌
   - 可切换查看：净资产 / 总资产 / 总负债
2. **趋势缩略图**：默认近30天曲线，点击进入完整分析
3. **资产构成环形图**：各类占比
4. **账户分类列表**：按类型分组，显示各账户当前值+上次变化
5. **悬浮按钮**：快速更新 / 添加账户

#### 3.2 状态设计
| 状态             | 展示                                    |
| ---------------- | --------------------------------------- |
| 无账户（新用户） | 引导页+"添加第一个账户"大按钮           |
| 仅1次记录        | 趋势图显示单点+提示"再次更新以查看趋势" |
| 长期未更新       | 顶部Banner"距上次更新已X天，去更新"     |

### 模块4：统计分析【P0图表 / P1高级指标】

#### 4.1 趋势分析
| 功能     | 规则                                |
| -------- | ----------------------------------- |
| 时间维度 | 30天/90天/1年/全部，分段切换        |
| 曲线类型 | 净资产（默认）/总资产/总负债        |
| 数据点   | 每个Snapshot为一个点，hover显示详情 |
| 里程碑   | 在曲线对应日期显示图标，点击查看    |
| 空数据期 | 两快照间用直线连接（线性插值展示）  |

#### 4.2 关键指标卡【P1】
| 指标          | 计算公式                                           |
| ------------- | -------------------------------------------------- |
| 当前净资产    | 见2.3                                              |
| 历史最高/最低 | max/min(净资产快照) + 日期                         |
| 平均月增长率  | 几何平均，(末值/初值)^(1/月数)−1                   |
| 投资真实收益  | Σ(投资类current_amount − total_invested)           |
| 收益拆分      | 区分"净流入增长"与"市值收益增长"（需累计投入数据） |

#### 4.3 分类分析
- 各资产类别趋势叠加对比
- 单账户独立趋势查看

## 四、关键状态流转

### 4.1 账户生命周期
```
[创建] → active ⇄ [更新快照] 
              ↓
         [归档] archived（保留历史，不计入当前净资产）
              ↓
         [删除]（二次确认，连带删除快照）
```

### 4.2 数据完整性规则
- 删除账户：弹窗警告"将删除X条历史记录"，需二次确认
- 推荐"归档"替代删除，归档账户在趋势图历史段仍参与计算
- 修改账户类型：不允许（避免数据混乱），需删除重建

## 五、非功能性需求

| 类别 | 要求                                            |
| ---- | ----------------------------------------------- |
| 性能 | 千级快照数据下，图表渲染<500ms                  |
| 安全 | 本地数据库加密（SQLCipher）；金额数据脱敏存储   |
| 隐私 | 不索取通讯录/位置；不爬取账户密码；明示隐私政策 |
| 兼容 | iOS 14+ / Android 9+                            |
| 离线 | 全功能离线可用，云同步为可选项                  |
