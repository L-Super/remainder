# 资产管理软件产品设计方案

## 一、产品定位

**核心价值**：帮助用户轻松追踪资产变化，直观呈现财富增长趋势，无需复杂记账操作。

**目标用户**：
- 希望了解整体资产状况但不想详细记账的用户
- 有多个账户/投资渠道需要统一管理的用户
- 关注长期财富增长而非日常开支的用户

**设计原则**

- 极简操作：最多3步完成任意操作
- 用户自由：支持机构名称由用户自定义，不预设限制
- 趋势优先：弱化明细，强化增长可视化

## 二、核心功能设计

### 2.1 资产记录

**资产大类与字段映射表**

| 资产类型   | 必填字段      | 选填字段                   | 净资产计算                 |
| ---------- | ------------- | -------------------------- | -------------------------- |
| 🏦 银行存款 | 机构名/余额   | 别名/卡类型/备注           | 储蓄卡计正，信用卡欠款计负 |
| 📈 股票账户 | 券商名/总资产 | 市场/累计投入/现金持仓拆分 | 总资产计正                 |
| 💰 基金账户 | 平台名/总市值 | 累计投入/类型分布          | 总市值计正                 |
| 🏠 固定资产 | 名称/估值     | 取得成本/关联负债          | 估值-负债                  |
| 🔒 保险理财 | 名称/现金价值 | 已交保费/到期金额          | 现金价值计正               |
| 💳 负债     | 名称/金额     | 利率/到期日                | **计负**                   |
| ➕ 自定义   | 名称/金额     | 正负属性/备注              | 用户指定正负               |

> **净资产 = Σ资产 − Σ负债**，这是财富追踪的核心数字。

多币种支持：

- 美股/港股账户可标记原币种

- 自动按汇率折算为本位币

### 2.2 趋势分析展示
**可视化图表**
- 图表展示资产总额变化
- 支持时间维度切换（近30天/90天/1年/全部）
- 显示增长率和增长金额
- 饼图展示各类资产占比
- 各资产类别增长曲线叠加对比

**关键指标**
- 当前总资产
- 较上次记录的变化（金额+百分比）
- 历史最高/最低资产值
- 支持单独查看某一账户的历史趋势
- 平均月增长率（年化收益参考）

> **指标计算口径需明确**：
>
> - "增长率"需区分「净流入增长」与「投资收益增长」。建议利用"累计投入"字段，估算 **真实收益 = 当前市值 − 累计投入**，避免把"存进去的钱"误算成"赚的钱"。

### 2.3 里程碑记录
**重要节点标记**
- 用户可标记特殊事件（买房、投资收益、奖金等）
- 在趋势图上显示标记点
- 帮助理解资产波动原因

### 2.4 资产更新机制

**自动快照**

- 适合周/月度统一复盘场景
- 自动生成一个"资产快照点"

**数据模型说明**

```
每次更新 = 一条快照记录(Snapshot)
Snapshot {
  accountId, amount, timestamp, note, milestoneTag?
}
趋势图 = 所有快照点连线
```



## 三、产品架构

### 信息架构
```
首页（资产总览）
├── 净资产卡片（核心数字+变化）
├── 增长趋势图
├── 资产/负债占比环形图
└── 账户分类列表
└── [快速更新] 悬浮入口

统计分析
├── 时间维度分析
├── 增长率统计（净流入 vs 收益拆分）
├── 分类占比
├── 分类资产列表及各分类趋势
└── 里程碑回顾
└── 历史记录

记录页面
├── 添加新账户（3步）
├── 单账户/批量更新
└── 账户编辑/删除/归档

设置
├── 提醒设置（系统智能提醒或用户自定义提醒周期，提醒录入）
├── 数据导出（CSV/Excel）
└── 数据备份与恢复（本地/云端）
```
